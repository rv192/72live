1.  [Droid CLI 发布 v0.23.0 更新并强调高效提示词缓存-Ben Tossell](https://x.com/bentossell/status/1988142685538918809)

    Droid CLI 发布了 **v0.23.0 版本更新**，带来了多项新功能和错误修复。新功能包括 **Markdown 表格渲染**，改进了结构化数据的显示；**“固定待办计划”**，允许用户在长时间会话中保持重要任务可见；以及 **MCP 工具结果格式优化**，提供更清晰易读的输出。同时，新版本修复了聊天完成 API 的思考块支持、基线版本回滚问题、droid exec 命令中的自治模式处理，并恢复了 Windows 上的 Ctrl+T 快捷键功能。此外，Droid 还被提及在处理**所有提供商的提示词缓存方面表现出色**，通常能达到 9:1 的高缓存率，显著提高了效率。

2.  [Kimi K2 训练团队在 Reddit LocalLLaMA 频道进行 AMA 问答精选-歸藏(guizang.ai)](https://x.com/op7418/status/1988109637078646883)

    Kimi K2 训练团队在 Reddit LocalLLaMA 频道进行了一次 AMA（Ask Me Anything）活动，回答了诸多技术和产品问题。团队透露，**KDA 混合结构**在预训练和强化学习阶段比传统方法更快速、经济、高效，可能用于 **K3 模型的训练**。K2 的**视觉语言（VL）版本正在开发中**。关于模型“非迎合式”人格设计，团队表示是通过精心挑选数据在预训练和后训练阶段共同塑造的。针对 API 计费透明度问题，团队承认用户困惑并考虑改进。他们选择 int4 量化是为了更好地兼容非 Blackwell GPU。K2-Thinking 模型虽然推理耗时，但正在优化，其在评估中已能捕捉 GPT-5 Sonnet 4.5 和 Opus 4.1 遗漏的问题。团队还提到，支持**交错的“思考-工具-思考-工具”模式**是开发 K2-Thinking 的最大挑战之一，并强调评估方法和数据质量对性能至关重要。训练硬件主要采用带有 Infiniband 的 H800 GPU。

3.  [Claude Sub Agent 被誉为最佳提示词管理工具-向阳乔木](https://x.com/vista8/status/1988109265312104631)

    有观点指出，**Claude 的 Sub Agent** 是目前**最好的提示词管理工具**。通过将常用的 AI 编程提示词直接创建为 Sub Agent，用户可以通过自然语言对话与模型交互，实现提示词的自动化调用。这种方式极大地简化了提示词的管理和使用流程，提高了工作效率。创建 Sub Agent 的过程简单直观：在 Claude 中输入 `/agents` 命令，选择“Create new agent”，并将其设置为个人作用域（`~/.claude/agents/`），然后输入名称、描述和提示词即可全局生效。

4.  [文心 App 推出“魔法漫画”功能，实现AI生成连续剧情漫画-向阳乔木](https://x.com/vista8/status/1988102787025240502)

    文心 App 近期推出了一项创新功能——**“魔法漫画”**，使用户能够通过一句话剧情描述，**在短时间内生成多页连续剧情漫画**。这项功能支持多种漫画风格选择，包括“真人电影”、“吉ト力”（吉卜力）和“3D 卡通”等。用户不仅可以利用内置的提示词模板，如“霸道总裁”、“穿越到十年后”等，还可以上传一张或多张真人照片，生成写实风格的剧情故事漫画，且人物样貌保持良好。该功能突破了传统漫画创作的门槛，让普通用户也能轻松实现故事的视觉化呈现。

5.  [OpenRouter AI 使新 LLM 及提供商即时集成至 Replit 应用-Amjad Masad](https://x.com/amasad/status/1988097679868801038)

    OpenRouter AI 宣布，其平台现已实现让**所有新的大型语言模型（LLM）和提供商能够即时在 Replit 应用中工作**。这一集成显著降低了开发者在 Replit 环境中访问和利用最新 AI 模型的技术壁垒，**极大地提升了开发效率和灵活性**。开发者可以更快地实验、构建和部署基于不同 LLM 的应用，无需复杂的配置过程，从而加速了 AI 创新和应用的普及。
