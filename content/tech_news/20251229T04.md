3.  [Claude Code在AI辅助编程领域实现飞跃 - 宝玉](https://x.com/dotey/status/2005069751920337194)

    Claude Code的创建者Boris Cherny披露，在过去一个月中，AI驱动的Claude Code结合Opus 4.5模型，完成了惊人的**259个PR**和**497次提交**，涉及近**8万行代码变动**，全部由AI生成。与一年前相比，其运行时间从几秒延长至数天，标志着AI编程工具的巨大进化。这一进展预示着“代码不再是瓶颈”，软件工程的重心正从**“执行”向“思考”**转移。

4.  [Claude Code赋能智能家居系统 - Andrej Karpathy](https://x.com/karpathy/status/2005067301511630926)

    Andrej Karpathy展示了Claude Code在智能家居领域的实际应用，成功接入Lutron家庭自动化系统。AI能够自动发现并连接控制器，识别设备固件，通过互联网获取系统手册，并指导用户完成设备配对。更令人兴奋的是，Claude Code甚至能直接控制厨房灯光，显著提升了家庭自动化体验，有望取代传统笨拙的iOS应用。

5.  [Claude Code Now回归极致简单：移除API切换器 - Orange AI](https://x.com/oran_ge/status/2005066598978302404)

    Claude Code Now的开发者反思了产品设计，决定在v1.6.1版本中**移除API切换器功能**，删除了885行代码。此举旨在回归其核心产品哲学——**极致简单**，确保用户能够专注于在**3秒内启动Claude Code**，而无需为API切换等复杂功能分心。这体现了开发者对产品核心价值的重新聚焦。

6.  [AI驱动的命令行视频处理工具ezff - 向阳乔木](https://x.com/vista8/status/2004948184565776386)

    一款名为ezff的命令行工具被推出，允许用户通过自然语言命令处理视频，**无需记忆复杂的ffmpeg指令**。该工具支持视频格式转换、压缩、分辨率调整、倍速修改及音频提取等多种操作。此外，ezff已被整合为Claude Code的Skill，进一步扩展了AI在视频后期制作中的应用，例如自动加配乐和截图。
